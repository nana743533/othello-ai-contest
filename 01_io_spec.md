# CodinGame Othello コンテスト入出力仕様書

## 1. ゲームの概要

このコンテストは、8x8の盤面で行われる標準的なオセロ（リバーシ）ゲームのAIを作成するものです。プレイヤーは黒（先手、Player 0）または白（後手、Player 1）を担当し、最終的により多くの石を盤面に残したプレイヤーが勝利となります。

## 2. 入力仕様

ゲームはターン制で進行し、毎ターン標準入力（`stdin`）からゲームの状態が与えられます。

### 2.1. 初回入力（ゲーム開始時のみ）

ゲームの最初のターン（ターン1およびターン2）でのみ、以下の2行が入力されます。

1. `id` (整数): プレイヤーのID（`0` が黒/先手、`1` が白/後手）
2. `boardSize` (整数): 盤面のサイズ（常に `8`）

### 2.2. 毎ターンの入力

毎ターン、以下の順序で入力が与えられます。

1. **盤面の状態**: `boardSize` 行の文字列。各行は盤面の1行を表し、上から下へ（y=0 から y=7）の順で与えられます。
   - `.` : 空のマス
   - `0` : 黒の石（Player 0）
   - `1` : 白の石（Player 1）

2. **Expert Modeの追加入力**（条件付き）:
   - プレイヤーが直前のターンで `EXPERT` コマンドを出力していた場合、ここで1行の文字列が追加で入力されます。
   - この文字列には、相手が直前のターン（またはパスが続いた場合は複数ターン）で行った手が含まれます。
   - 複数の手がある場合は `;` で区切られます（例: `d3;pass;`）。

3. **合法手の数**: `actionCount` (整数)
   - 現在のターンでプレイヤーが打つことができる合法手の数。

4. **合法手のリスト**: `actionCount` 行の文字列
   - 各行に1つの合法手の座標（例: `d3`, `f5`）が与えられます。
   - 座標は `a`〜`h`（列、左から右）と `1`〜`8`（行、上から下）の組み合わせです。

## 3. 出力仕様

毎ターン、標準出力（`stdout`）に1行で行動を出力する必要があります。

### 3.1. 基本的な出力フォーマット

石を置きたいマスの座標を出力します。

```
<座標>
```
例: `d3`

### 3.2. メッセージの追加

行動の後に `MSG` キーワードを付けることで、ゲーム画面にメッセージを表示させることができます（最大12文字）。

```
<座標> MSG <メッセージ>
```
例: `d3 MSG Hello!`

### 3.3. Expert Modeの有効化

出力の先頭に `EXPERT` キーワードを付けることで、次ターンから相手の行動履歴を受け取る「Expert Mode」を有効にできます。

```
EXPERT <座標>
```
例: `EXPERT d3` または `EXPERT d3 MSG Hello!`

## 4. 制約条件とルール

### 4.1. タイムリミット
- **初回ターンの応答時間**: 2000ミリ秒（2秒）以内
- **2ターン目以降の応答時間**: 150ミリ秒（0.15秒）以内
- タイムアウトした場合、そのプレイヤーは敗北となります。

### 4.2. ゲームの進行ルール
- 黒（Player 0）が常に先手です。
- 合法手がない場合、プレイヤーのターンは自動的にスキップ（パス）されます。
- 両プレイヤーともに合法手がない場合、ゲームは終了します。
- 出力した座標が盤面外であるか、すでに石が置かれているマスであるか、または合法手でない場合、不正な行動（Invalid Action）となり即座に敗北します。

### 4.3. 勝利条件
- ゲーム終了時に、相手より多くの石を持っていること。
- 相手の石を0個にした場合、その時点で勝利となります。
- 相手がタイムアウトまたは不正な行動をした場合も勝利となります。
- 石の数が同じ場合は引き分け（Draw）となります。
